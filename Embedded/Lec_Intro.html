Embedded Systems: Information and Basicss

Lecture Overview

What is a System?

A system is a way of working, organizing, or performing one or many tasks according to a fixed set of Rules and Parts.

Embedded System Definition: A combination of computer hardware and software, either fixed in capability or programmable, designed for a specific function.

The Comparison

General Purpose Computer

  • Large Memory (RAM/ROM)
  • Keyboard, Mouse, Monitor
  • Designed for many different tasks

Embedded System

  • Hardware Processor & Timers
  • Main Application Software
  • Real-Time Operating System (RTOS)

The Processor: The Heart of the System

For an embedded designer, knowledge of microprocessors and microcontrollers is essential. The processor consists of two essential units:

  • Control Unit (CU): Manages the "Fetch" operations.
  • Execution Unit (EU): Manages the "Execute" operations.

Other Key Hardware Components

  • Power Source & Clock Oscillator
  • Real-Time Clock (RTC) & Watchdog Timers
  • Memory (RAM/ROM)
  • I/O Ports & Interrupt Handlers

Embedded System Constraints

Unlike standard IT environments, we must design around three strict limits:

  1. Available System Memory: Often very small (KB or MB).
  2. Processor Speed: Optimized for efficiency, not raw power.
  3. Power Dissipation: Systems must often run cool or on battery power.

Software & RTOS

Embedded software often uses a Real-Time Operating System (RTOS). This sets the rules for execution and ensures the system handles multiple tasks concurrently without failing timing requirements.

Where do we see these?

Embedded systems are everywhere in our modern world:

  • Household: Microwaves, TVs, Washing Machines.
  • Medical: Equipment and monitoring devices.
  • Automotive: Engine controllers and braking systems.
  • Security: Smart locks and firewalls.

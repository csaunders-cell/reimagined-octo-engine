Web Development 101: Your Interactive Tutorial

Tutorial Steps

1. Semantic HTML: The Skeleton

Think of HTML (HyperText Markup Language) as the structure, the skeleton of your web page. We use elements (tags) like boxes to define content. When you work with a CMS, you might insert content into a 'visual editor', but the CMS is generating these tags behind the scenes.

For example, <h1> is a 'Heading 1', and <p> is a 'Paragraph'. Using these correctly is essential for both accessibility and search engine optimization (SEO).

Your Task: In the 'HTML' box on the right, try changing the text inside the <h1> tag and the <p> tag. Can you add a link using an anchor tag <a> like this: <a href="https://google.com">Go to Google</a>? Click 'Run Code' to see the changes below.

2. Basic CSS: The Style

If HTML is the skeleton, CSS (Cascading Style Sheets) is the skin and clothing. It tells the browser *how* the HTML elements should look—their color, font, size, and spacing. When you change a 'theme' or 'template' in a CMS, you are swapping out a whole set of CSS rules.

CSS works with rulesets. For instance: body { color: blue; } makes all text on the page blue.

Your Task: Switch to the 'CSS' tab. Change the color of the h1 header from `#2c3e50` to `red` or `#e74c3c`. Try changing the `background-color` of the `body`. Try adjusting the `padding` and `margin`. This controls the empty space around your boxes!

3. JavaScript: The Behavior

JavaScript is the 'muscle'. It adds logic and allows the page to respond to actions, like a user clicking a button. When you add dynamic features, animations, or live filters in a CMS using plugins, you are often relying on complex JavaScript libraries.

JavaScript interacts with the DOM (Document Object Model), a structured map of your HTML. It allows the code to select and change specific boxes on the page.

Your Task: Switch to the 'JavaScript' tab. Can you change the behavior so that the background color becomes `purple`? Try changing the message in the `alert()`. Can you make the JavaScript change the `innerHTML` of the `h1` header instead?

Conclusion: Prepping for a CMS

You’ve built a simple page from scratch using the fundamental triad of web tech: HTML structure, CSS style, and JavaScript behavior.

When you later work with a content management system (CMS) like WordPress or Webflow, remember: these platforms are just automated, often drag-and-drop systems for generating the precise code you’ve seen today. The concepts remain the same: you define content types (HTML), style templates (CSS), and perhaps dynamic features (JavaScript/Plugins). Your foundational knowledge will help you troubleshoot and customize far more effectively!
